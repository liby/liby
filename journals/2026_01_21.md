- Prettier VS Code 扩展的“折腾史”
	- 1. 两个月前：VS Code 提示 `esbenp.prettier-vscode` 已 deprecated，要迁移到 `Prettier.prettier-vscode`
	  2. 今天发现：官方又换回了 `esbenp.prettier-vscode`
	  3. 反转原因：
		- 新扩展的 publisher 命名空间注册成了大写 Prettier
		- Marketplace 显示扩展 ID：`Prettier.prettier-vscode`（含大写）
		- 但 VS Code 配置里 formatter 只能写小写：`prettier.prettier-vscode`
		- 用户复制 Marketplace 的 ID 就会配置失败/混乱
	- 4. 结局：
		- publisher 命名空间大小写无法修改
		- 修复成本过高（需要删 publisher 重建 + 影响大量用户配置）
		- 官方放弃新扩展，退回 `esbenp.prettier-vscode`
	- 坑点 / 经验
		- VS Code 内部会把扩展 ID 统一为小写，但 Marketplace 会保留原始大小写
		- publisher 大小写注册错了几乎不可修复
		- 扩展 ID 会被用户写死到 `settings.json` / `extensions.json`，迁移代价巨大
